Documentación de API - Inventario Facilito


Usuarios

Descripción

Crea un nuevo usuario en la base de datos.

Parámetros
  • Usuario: Nombre del usuario.
  • Contrasena: Contraseña del usuario.
  • Correo: Correo electrónico del usuario.
Ejemplo de uso
          
  POST /Registrar
  Content-Type: application/json
  
  {
    "Usuario": "ejemploUsuario",
    "Contrasena": "ejemplo123",
    "Correo": "usuario@example.com"
  }
Respuestas
  • 200 OK: Usuario creado exitosamente.
  • 401 Unauthorized: Error si el usuario ya existe o hay algún otro problema.
  • 500 Internal Server Error: Error interno del servidor.
Descripción

Autentica a un usuario comparando el nombre de usuario y contraseña proporcionados con los registros de la base de datos.

Parámetros
  • Usuario: Nombre de usuario.
  • Contrasena: Contraseña del usuario.
Ejemplo de uso
          
  POST /Login
  Content-Type: application/json
  
  {
    "Usuario": "ejemploUsuario",
    "Contrasena": "ejemplo123"
  }
Respuestas
  • 200 OK: Si la autenticación es exitosa. Retorna el nombre de usuario y la contraseña.
  • 401 Unauthorized: Si las credenciales proporcionadas son incorrectas o el usuario no existe.
  • 500 Internal Server Error: Error interno del servidor.

Bodegas

Descripción

Obtiene las bodegas asociadas a un usuario específico junto con las etiquetas asociadas a cada bodega.

Parámetros
  • Correo_usuario: Correo electrónico del usuario cuyas bodegas se quieren obtener.
Ejemplo de uso
          
  GET /Obtener_Bodegas?Correo_usuario=usuario@example.com
Respuestas
  • 200 OK: Retorna un arreglo de objetos donde cada objeto contiene el nombre de la bodega y las etiquetas asociadas.
  • 401 Unauthorized: Si el usuario proporcionado no es válido o no tiene acceso.
  • 500 Internal Server Error: Error interno del servidor.
Descripción

Crea una nueva bodega en la base de datos y la asocia al usuario especificado junto con las etiquetas proporcionadas.

Parámetros
  • Nombre_bodega: Nombre de la nueva bodega a crear.
  • Correo_usuario: Correo electrónico del usuario que será administrador de la bodega.
  • Lista_tags: Arreglo de IDs de etiquetas que se asociarán a la nueva bodega.
Ejemplo de uso
          
  POST /Crear_bodega
  Content-Type: application/json
  
  {
    "Nombre_bodega": "Ejemplo Bodega",
    "Correo_usuario": "usuario@example.com",
    "Lista_tags": [1, 2]
  }
Respuestas
  • 200 OK: Bodega creada y asociada correctamente.
  • 401 Unauthorized: Si el usuario proporcionado no es válido.
  • 500 Internal Server Error: Error interno del servidor al crear la bodega o asociar etiquetas.
Descripción

Crea una nueva etiqueta en la base de datos asociada al usuario especificado.

Parámetros
  • Nombre_tag: Nombre de la nueva etiqueta a crear.
  • Correo_usuario: Correo electrónico del usuario que crea la etiqueta.
Ejemplo de uso
          
  POST /Crear_tag
  Content-Type: application/json
  
  {
    "Nombre_tag": "Electrónica",
    "Correo_usuario": "usuario@example.com"
  }
Respuestas
  • 200 OK: Etiqueta creada correctamente.
  • 401 Unauthorized: Si el usuario proporcionado no es válido.
  • 500 Internal Server Error: Error interno del servidor al crear la etiqueta.
Descripción

Obtiene las etiquetas creadas por un usuario específico.

Parámetros
  • Correo_usuario: Correo electrónico del usuario del cual se desean obtener las etiquetas.
Ejemplo de uso
          
  GET /Obtener_tags?Correo_usuario=usuario@example.com
Respuestas
  • 200 OK: Retorna un arreglo de objetos JSON con los nombres de las etiquetas del usuario.
  • 401 Unauthorized: Si el usuario proporcionado no es válido o no tiene acceso.
  • 500 Internal Server Error: Error interno del servidor.

Articulos

Descripción

Obtiene los productos almacenados en una bodega específica.

Parámetros
  • Bodega_Nombre: Nombre de la bodega de la cual se desean obtener los productos.
Ejemplo de uso
          
  GET /Obtener_articulos?Bodega_Nombre=Bodega1
  Content-Type: application/json
Respuestas
  • 200 OK: Lista de productos obtenidos correctamente.
  • 500 Internal Server Error: Error interno del servidor al intentar obtener los productos.
Descripción

Crea un nuevo producto en la base de datos con los detalles proporcionados.

Parámetros
  • Nombre: Nombre del producto.
  • Descripcion: Descripción del producto.
  • Imagen: URL o base64 de la imagen del producto.
  • Precio: Precio del producto.
  • Codigo: Código único del producto.
Ejemplo de uso
          
  POST /Crear_Producto
  Content-Type: application/json
  
  {
    "Nombre": "Producto1",
    "Descripcion": "Descripción del producto 1",
    "Imagen": "https://ejemplo.com/imagen1.jpg",
    "Precio": 100.50,
    "Codigo": "ABC123"
  }
Respuestas
  • 200 OK: Producto creado correctamente.
  • 500 Internal Server Error: Error interno del servidor al intentar crear el producto.
Descripción

Obtiene la información de stock de un producto específico en una bodega específica.

Parámetros
  • Producto_nombre: Nombre del producto del cual se desea obtener el stock.
  • Bodega_Nombre: Nombre de la bodega donde se encuentra el producto.
Ejemplo de uso
          
  GET /Obtener_Stock?Producto_nombre=Producto1&Bodega_Nombre=Bodega1
Respuestas
  • 200 OK: Retorna un arreglo de objetos JSON con los detalles del stock del producto en la bodega.
  • 500 Internal Server Error: Error interno del servidor al intentar obtener el stock.
Descripción

Añade stock de un producto a una bodega específica.

Parámetros
  • Cantidad: Cantidad del producto a añadir al stock.
  • Fecha_vencimiento: Fecha de vencimiento del lote añadido.
  • Fecha_ingreso: Fecha de ingreso del lote añadido.
  • Producto_nombre: Nombre del producto al cual se añadirá el stock.
  • Bodega_Nombre: Nombre de la bodega donde se añadirá el stock.
Ejemplo de uso
          
  POST /Añadir_Stock
  Content-Type: application/json
  
  {
    "Cantidad": 100,
    "Fecha_vencimiento": "2025-12-31",
    "Fecha_ingreso": "2024-06-15",
    "Producto_nombre": "Producto1",
    "Bodega_Nombre": "Bodega1"
  }
Respuestas
  • 200 OK: Stock añadido correctamente a la bodega.
  • 500 Internal Server Error: Error interno del servidor al intentar añadir el stock.
Descripción

Edita los detalles de un producto existente.

Parámetros
  • ID: ID del producto que se desea editar.
  • Nombre: Nuevo nombre del producto.
  • Descripcion: Nueva descripción del producto.
  • Imagen: Nueva URL de la imagen del producto.
  • Precio: Nuevo precio del producto.
  • Codigo: Nuevo código del producto.
Ejemplo de uso
          
  PUT /Editar_Producto
  Content-Type: application/json
  
  {
    "ID": 1,
    "Nombre": "Nuevo Nombre",
    "Descripcion": "Nueva descripción",
    "Imagen": "https://ejemplo.com/nueva-imagen.jpg",
    "Precio": 15000,
    "Codigo": "987654"
  }
Respuestas
  • 200 OK: Producto editado correctamente.
  • 500 Internal Server Error: Error interno del servidor al intentar editar el producto.
